# 每日快報流程交接契約

## 流程總覽

`新聞蒐集 → 事件整併與排序 → 分類與摘要 → 客群內容 → 驗證與封裝`

## 階段交接

| 階段 | 執行 Skill | 必要輸入 | 主要輸出 | 不負責 |
|---|---|---|---|---|
| 新聞蒐集 | `collect-approved-news` | 日期、窗口、主題、白名單 | 文章層 raw data | 合併、摘要、排序 |
| 事件整併與排序 | `group-rank-news-events` | 候選文章、評分設定 | 事件群組、來源、焦點分數 | 最終分類、公開摘要 |
| 分類與摘要 | `classify-summarize-news` | 已排序事件、來源、Schema | 網頁事件資料 | 每日概況、話術 |
| 客群內容 | `generate-audience-content` | 已驗證事件、主題、客群 | 每日概況、主題概況、話術 | 改動新聞事實 |
| 驗證與封裝 | `validate-package-daily-data` | 所有階段結果、Schema | 每日 JSON、索引、驗證報告 | AI 補值、GitHub 發布 |

## 交接原則

- 每階段只修改自己負責的欄位。
- 原始來源欄位在所有階段保持可追溯。
- 後續階段發現上游問題時，退回對應 Skill 修正，不在下游偷偷補值。
- `warnings` 與 `manual_review_items` 必須一路保留。
- 重新執行單一階段時，確認上游輸入版本沒有改變。

## 可重跑範圍

- 來源白名單或搜尋窗口改變：從新聞蒐集重跑。
- 事件合併或焦點權重改變：從事件整併與排序重跑。
- 主題邊界或摘要規則改變：從分類與摘要重跑。
- 客群或話術規則改變：只重跑客群內容及驗證。
- Schema 或輸出格式改變：重跑驗證與封裝，必要時遷移既有日期。

## 執行報告

最後至少列出：

- 每個主題的候選文章數與事件數。
- 多來源事件數。
- 無法存取、時間不明及來源衝突項目。
- 各主題焦點分數範圍。
- 驗證結果及人工檢查清單。
- 本次實際變更檔案。
