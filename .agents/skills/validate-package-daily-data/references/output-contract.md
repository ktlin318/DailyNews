# 每日資料輸出契約

## 唯一標準

`data/schema.json` 是網頁資料格式的機器可讀契約；`config/taxonomy.json` 是主題與客群的唯一設定來源。主題或客群有變動時，只修改 Taxonomy，再重建每日資料並執行驗證；欄位結構變動時才修改 Schema。

## 每日檔案

`data/YYYY-MM-DD.json` 必須包含：

- Schema 版本、日期、時區、產生時間與來源窗口。
- 每日概況及所有主題概況。
- 主題與客群陣列。
- 已排序事件。
- 所有主題與客群組合的話術。
- 警告及人工檢查項目。

每個事件必須包含穩定 ID、主要主題、分類理由、時間、中立標題、摘要、影響、關鍵字、客群、完整來源、證據註記及焦點評分資料。

## 日期索引

`data/index.json`：

- `latest` 指向最新可用日期。
- `dates` 由新到舊排列。
- 只有每日檔通過驗證後才能加入。
- 不存在資料檔的日期不得出現在索引。

## Raw data

- 保存在 `rawdata/YYYY-MM-DD.raw.json`。
- 可包含清理後文章內容、蒐集狀態、事件群組及評分工作資料。
- 網頁每日檔只放摘要與來源，不放完整文章正文。

## 網頁呈現相關規則

- 同一事件的多個來源都保留為獨立連結。
- 顯示日期採最早合格來源的月日。
- 事件依焦點分數排序。
- 每頁最多十則由前端處理，資料層不為分頁刪除事件。
