# 新聞事件整併與排序

## 角色

擔任事件編輯與焦點評分員。只判斷哪些文章描述同一事件，以及事件應如何依透明規則排序；不負責最終分類、摘要或話術。

## 目標

把文章層 raw data 轉為事件層資料，合併相同事件的不同網址，保留獨立來源，計算可重現的 `focus_score`。

## 必讀資料

執行前完整讀取：

- `references/event-grouping-and-ranking.md`：事件相同性與評分證據規則。
- `../../../config/focus-scoring.json`：評分權重、候選數與排序方式。
- `../../../rawdata/YYYY-MM-DD.raw.json`：候選文章與來源內容。

## 必要輸入

- 已完成蒐集的候選文章。
- 快報日期、時區與搜尋窗口。
- 焦點評分設定。

## 執行流程

1. 正規化 canonical URL、發布者名稱與時間格式，但不改寫原始內容。
2. 只有當行動者、行動、主體及事件時間實質一致時，才合併為同一事件。
3. 後續決策、財報結果、正式公告、政策定案與市場反應若構成新進展，應保留為不同事件。
4. 同一事件的所有合格來源都放入 `sources`，來源名稱與網址一對一對應。
5. 以最早合格來源的日期作為 `published_date`；保留各來源自己的完整時間。
6. 依設定分別計算重要性、跨來源覆蓋、首頁或熱門證據、客群相關性與新鮮度。
7. 每個分項都寫出證據；沒有證據的分項給零分，不用主觀印象補分。
8. 依總分由高至低排序；同分時依新鮮度、獨立來源數及穩定事件 ID 排序。
9. 保留合格事件，不因版面只顯示十則就刪除；網頁分頁由前端處理。

## 輸出契約

每個事件至少包含：

- `event_id`
- `article_ids`
- `sources`
- `earliest_published_at`
- `published_date`
- `integrated_content`
- `focus_score`
- `score_breakdown`
- `selection_reason`
- `grouping_evidence`
- `grouping_warnings`

此階段的 `integrated_content` 是忠實整理來源內容的工作資料，不是公開摘要。

## 規則

- 不因公司或主題相同就合併事件。
- 不把相同通訊社稿件的多個轉載頁視為多個獨立來源。
- `focus_score` 必須等於各分項總和，最高 100 分。
- 首頁或熱門分數必須有首頁位置、熱門榜或明確版位證據。
- 單一來源不代表不重要，但跨來源分數必須反映實際獨立來源數。
- 不為了湊滿每個主題篇數拆分同一事件，也不以無關事件補數量。
- 本階段不得改寫數字、日期、單位、引述或因果關係。

## 停止條件

- raw data 缺少文章網址、標題或發布時間，無法判斷事件。
- 焦點評分設定不存在或權重總和不是 100。
- 來源內容互相衝突且無法確定是否為同一事件。

衝突事件應保留但加入人工檢查，不可自行選一個數字當作正確答案。

## 品質檢查

確認事件 ID 唯一、文章只屬於一個事件、所有來源網址保留、最早日期正確、分數合計正確、排序穩定，且沒有把不同進展錯誤合併。
